Cloud Migrations & Hybrid WANs

Designing and deploying wide-area networks that move data to hybrid cloud environments without relying solely on big cloud providers.

Back to Timeline

The Philosophy

Not every business needs or wants AWS, Azure, or Google Cloud. Some need control. Some need compliance. Some just need their data to stay in Canada. I design hybrid infrastructure that uses cloud where it makes sense and keeps critical systems on-premise or in private data centers where it matters.

What I Deliver

  • Hybrid WAN design — connecting branch offices to central infrastructure and selective cloud services
  • VPN and SD-WAN deployments for secure, redundant connectivity
  • Data migration planning — what moves, what stays, and how to move it without downtime
  • Private cloud builds using Proxmox, VMware, and open-source orchestration
  • Cost modeling — comparing true cloud TCO vs. owned infrastructure over 3-5 years

Typical Architecture

Core Infrastructure

On-premise virtualization cluster (Proxmox/VMware) running Active Directory, file servers, and legacy applications.

Edge Services

Cloud-hosted web front-ends, email, and collaboration tools connected via secure tunnels.

Backup & DR

Replicated backups to off-site storage with tested recovery procedures and documented RTO/RPO.

Technology

Proxmox / VMware Azure (Selective) SD-WAN IPSec / WireGuard PFSense / OPNsense Linux Networking TrueNAS / Ceph

What I Learned

The cloud isn't always cheaper. For predictable workloads with steady growth, owning your infrastructure often wins on a 5-year TCO. The real value of cloud is elasticity — burst capacity, global distribution, and managed services. My job is to help clients find the right mix, not sell them a one-size-fits-all solution.
